# Security Policy — AFRION Sahara

We take security seriously, especially because AI projects can be misused.

## Supported versions
This repository is early-stage. Security fixes will be prioritized on `main` until versioned releases exist.

## Reporting a vulnerability
**Do not open a public GitHub Issue** for security vulnerabilities.

**TBD:** Private reporting channel.
Recommended options:
1) security@afrion.ai (when domain/email is available)
2) GitHub Private Vulnerability Reporting (enable in repo settings)
3) A private Google Form for reports (temporary)

Until a channel is set:
- Use GitHub Discussions with minimal detail and request a private contact route.

## What to include
- Description of the issue
- Steps to reproduce
- Potential impact
- Any suggested fixes/patches

## Coordinated disclosure
- We will acknowledge receipt as soon as possible.
- We will coordinate disclosure timing to reduce risk.

## Sensitive content
Do not contribute anything that could enable:
- credential theft
- malware delivery
- targeted harassment/violence
- privacy violations
- bypassing safeguards

Those contributions will be rejected.
